# AI Gate Iraq User Guide

## 1. Open the platform

Visit the official AI Gate Iraq platform and choose Kurdish or Arabic from the language control. The interface uses right-to-left layout for both languages.

## 2. Sign in

1. Select **چوونەژوورەوە / تسجيل الدخول** from the sidebar.
2. Choose the approved Google account.
3. Complete the Google sign-in prompt.
4. Return to the platform after authentication succeeds.

Some public tools can be viewed without signing in. Account-based records and protected functions require authentication.

## 3. Start a new conversation

Select **چاتی نوێ / محادثة جديدة** from the sidebar. The current conversation is preserved according to the platform session behavior, and a clean assistant workspace is opened.

## 4. Choose a service

The sidebar contains the available business and logistics services:

- Smart business assistant
- Market and trade guidance
- Border information
- Currency conversion
- Cost estimation
- Business registration and KYC guidance
- Procurement and sourcing
- Shipment tracking
- Logistics map
- Public inquiry or demo request, when enabled

Choose only the service relevant to the task. Data entered in one workspace should be reviewed before using it for a legal, customs, or financial decision.

## 5. Use the smart assistant

1. Choose the relevant chat scope.
2. Enter a clear question with the product, route, country, currency, or business context.
3. Review the response and any stated limitations.
4. Verify legal, customs, price, and regulatory information with an official source or qualified adviser.

Do not submit passwords, payment-card data, private keys, or unnecessary personal data in chat.

## 6. Save and restore work

The session manager and sidebar panels can restore supported conversations and project context stored by the platform. Shared devices should be signed out after use. Clear browser data when sensitive local session history must be removed.

## 7. Conversion and pilot requests

Public website forms collect the fields required to evaluate the request. Submit accurate contact information so the support team can respond. Do not submit confidential commercial documents through a public text field.

## 8. Administrative users

Administrative screens are restricted to authenticated users with an active `adminUsers/{uid}` record and role `owner` or `admin`. Administrators must refresh authentication before privileged operations and must not share accounts.

## 9. Troubleshooting

### Sign-in does not open

- Allow pop-ups for the official platform.
- Confirm the browser is online and JavaScript is enabled.
- Try the approved Google account again.
- Send the displayed domain and error code to support without sending an authentication token.

### A service does not load

- Refresh the page once.
- Check the production status and network connection.
- Record the time, selected service, browser, and screenshot.
- Contact `support@idg.official.iq`.

### Permission denied

A permission error normally means the signed-in account is not authorized for the requested record or administrative action. Sign out and sign in with the approved account. Administrators should confirm the active admin document and role.

## 10. Support request format

Include:

- Name and company
- Contact email
- Affected website or platform URL
- Date and local time
- Browser and device
- Steps that reproduce the problem
- Screenshot with sensitive data hidden
- Error text or code

Support email: `support@idg.official.iq`
